Without Prerender

When you click the link, the browser starts from scratch: resolves DNS, opens a TCP connection, sends the request, waits for the server, parses HTML, loads CSS & JS, and finally paints.

Typical load: 300–2000ms

With Prerender

All of that work was already done while you were on the previous page. Clicking the link just swaps the hidden pre-rendered page into view — no waiting at all.

Typical activation: < 5ms

What the browser does during prerender
Yes Fetches the full HTML document
Yes Parses the DOM tree
Yes Loads and applies CSS stylesheets
Yes Executes JavaScript (with some restrictions)
Yes Renders the composited visual layer
Wait Does NOT display it — waits for activation (the click)
