Prerender — What happened before you clicked

Without Prerender

  • 1You click the link
  • 2Browser sends a network request
  • 3Server responds (TTFB)
  • 4HTML parsed, CSS & JS loaded
  • 5Render tree built, layout calculated
  • 6Page painted to screen

User waits: full network + render time

With Prerender (this page)

  • 1Speculation rule parsed → browser fetches HTML
  • 2Full parse + CSS + JS runs in hidden tab
  • 3Composited frame held in memory
  • 4You click the link
  • 5Browser activates the pre-rendered page
  • 6Page visible instantly

User waits: ~0ms (activation only)
